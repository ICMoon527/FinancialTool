# AGENTS.md

本文件定义在本仓库中执行开发、Issue 分析、PR 审查时的统一行为准则。

## 1. 通用协作原则

- 语言与栈：Python 3.10+，遵循仓库现有架构与目录边界。
- 配置约束：统一使用 `.env`（参见 `.env.example`）。
- 代码质量：优先保证可运行、可回归验证、可追踪（日志/错误信息清晰）。
- 测试文件管理：测试文件统一放入tests文件夹中，并在测试完毕后删除。
- 风格约束：
  - 行宽 120
  - `black` + `isort` + `flake8`
  - 关键变更需至少做语法检查（`py_compile`）或对应测试验证。
  - 新增或修改的代码注释必须使用中文。
- Git 约束：
  - 未经明确确认，不执行`git`相关操作
- 需要注意的问题：
  - 不要删除任何的配置文件比如.env，如有配置文件需要，优先级：当前子项目文件夹的配置文件中添加>在当前子项目文件夹中创建>总项目目录中的配置文件中添加>在总项目目录中创建
