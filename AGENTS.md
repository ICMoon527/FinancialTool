# AGENTS.md

本文件定义在本仓库中执行开发、Issue 分析、PR 审查时的统一行为准则。

## 1. 通用协作原则

- 语言与栈：Python 3.10+，遵循仓库现有架构与目录边界。
- 配置约束：统一使用 `.env`（参见 `.env.example`）。
- 代码质量：优先保证可运行、可回归验证、可追踪（日志/错误信息清晰）。
- 测试文件管理：测试文件统一放入tests文件夹中，并在测试完毕后删除。
- 风格约束：
  - 行宽 120
  - `black` + `isort` + `flake8`
  - 关键变更需至少做语法检查（`py_compile`）或对应测试验证。
  - 新增或修改的代码注释必须使用中文。
- Git 约束：
  - 未经明确确认，不执行`git`相关操作
- 关键变更需做类型检查 （`pyright`，内置编译器 `tsc`）, 结构化测试 `pytest`, Pre-commit Hooks`pre-commit`
- 把复杂任务拆成多个子任务，每个子 Agent 有自己的上下文窗口。
- 写完代码后，自己跑测试、看截图、查日志，如果修改了前端代码则通过浏览器确认前端效果是否符合预期。
- 需要注意的问题：
  - 不要删除任何的配置文件比如.env，如有配置文件需要，优先级：当前子项目文件夹的配置文件中添加>在当前子项目文件夹中创建>总项目目录中的配置文件中添加>在总项目目录中创建
  - 避免产生类似'XXX' object has no attribute 'XXX'的问题，完成检查后再结束任务
  - 避免产生Future Warning，可以选择过滤
  - 通过API联网获取数据时尽量获取完整的数据，避免获取部分数据导致数据不完整影响后续计算及分析
