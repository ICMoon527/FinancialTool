# 策略数据需求规范

## 概述
本文档详细说明了 `top_rise_stock_selector` 和 `dragon_leader` 两个选股策略的完整数据范围及具体数据类型要求。

---

## 一、top_rise_stock_selector（领涨选股策略）

### 1. 数据时间范围

| 数据类型 | 时间范围 | 说明 |
|---------|---------|------|
| **历史数据周期** | 至少 **60 个交易日** | 用于计算技术指标（如 EMA60、HHV60 等） |
| **推荐历史数据** | **120 个交易日** | 确保所有指标计算的稳定性 |
| **实时数据更新频率** | 每日收盘后更新 | 用于每日选股 |

### 2. 具体数据字段及类型

| 字段名 | 数据类型 | 必填 | 说明 | 来源示例 |
|--------|---------|------|------|----------|
| `code` | 字符串(String) | 是 | 股票代码（6位数字） | "600519", "000001" |
| `date` | 日期(Date) | 是 | 交易日期 | 2025-03-24 |
| `open` | 数值型(Float) | 是 | 开盘价 | 178.50 |
| `high` | 数值型(Float) | 是 | 最高价 | 182.30 |
| `low` | 数值型(Float) | 是 | 最低价 | 176.20 |
| `close` | 数值型(Float) | 是 | 收盘价 | 180.80 |
| `volume` | 数值型(Float/Int) | 是 | 成交量（单位：股） | 12580000 |
| `amount` | 数值型(Float) | 否 | 成交额（单位：元） | 2268000000.00 |
| `pct_chg` | 数值型(Float) | 否 | 涨跌幅（%） | 2.35 |

### 3. 数据来源及获取方式

| 数据源 | 优先级 | 获取方式 | 说明 |
|--------|--------|---------|------|
| **Tushare** | 1（最高） | `daily` 接口 | 数据质量高，接口稳定 |
| **Akshare** | 2 | 东方财富/新浪财经接口 | 备选数据源 |
| **Efinance** | 3 | 东方财富接口 | 备选数据源 |

### 4. 数据质量要求

#### 4.1 完整性要求
- **必填字段完整性**：`open`, `high`, `low`, `close`, `volume` 字段不能为空
- **日期连续性**：交易日数据应连续，不得缺少交易日（停牌除外）
- **数据覆盖度**：每只股票至少应有 60 个交易日的历史数据

#### 4.2 准确性要求
- **价格精度**：价格字段保留 2 位小数
- **成交量精度**：成交量为整数（单位：股）
- **逻辑一致性**：`high` ≥ `max(open, close)`，`low` ≤ `min(open, close)`

#### 4.3 时效性标准
- **更新频率**：每个交易日收盘后（约 17:00 后）更新当日数据
- **数据延迟**：T+1 数据应在 T+1 日 20:00 前完成更新
- **历史数据回溯**：首次使用时需下载至少 120 个交易日的历史数据

## 二、dragon_leader（龙头策略组合）

### 策略组成
该策略包含三个子策略：
1. **dragon_leader_start**（龙头启动策略）
2. **dragon_leader_main_rise**（龙头主升策略）
3. **dragon_leader_second_wave**（龙头二波策略）

### 1. 数据时间范围

| 数据类型 | 时间范围 | 说明 |
|---------|---------|------|
| **历史数据周期** | 至少 **135 个交易日** | 用于计算所有技术指标（最长周期为 135） |
| **推荐历史数据** | **200 个交易日** | 确保所有指标计算的稳定性和回溯测试 |
| **实时数据更新频率** | 每日收盘后更新 | 用于每日选股 |

### 2. 具体数据字段及类型

#### 2.1 基础 OHLCV 数据（所有子策略共用）

| 字段名 | 数据类型 | 必填 | 说明 | 来源示例 |
|--------|---------|------|------|----------|
| `code` | 字符串(String) | 是 | 股票代码（6位数字） | "600519", "000001" |
| `date` | 日期(Date) | 是 | 交易日期 | 2025-03-24 |
| `open` | 数值型(Float) | 是 | 开盘价 | 178.50 |
| `high` | 数值型(Float) | 是 | 最高价 | 182.30 |
| `low` | 数值型(Float) | 是 | 最低价 | 176.20 |
| `close` | 数值型(Float) | 是 | 收盘价 | 180.80 |
| `volume` | 数值型(Float/Int) | 是 | 成交量（单位：股） | 12580000 |
| `amount` | 数值型(Float) | 否 | 成交额（单位：元） | 2268000000.00 |

#### 2.2 各子策略额外需求

##### dragon_leader_start（龙头启动策略）
- **需要最长周期**：34 个交易日（RSV 指标）
- **需要额外字段**：无，基础 OHLCV 即可

##### dragon_leader_main_rise（龙头主升策略）
- **需要最长周期**：30 个交易日（均线指标）
- **需要额外字段**：无，基础 OHLCV 即可

##### dragon_leader_second_wave（龙头二波策略）
- **需要最长周期**：135 个交易日（量能均线）
- **需要额外字段**：无，基础 OHLCV 即可

### 3. 数据来源及获取方式

| 数据源 | 优先级 | 获取方式 | 说明 |
|--------|--------|---------|------|
| **Tushare** | 1（最高） | `daily` 接口 | 数据质量高，接口稳定 |
| **Akshare** | 2 | 东方财富/新浪财经接口 | 备选数据源 |
| **Efinance** | 3 | 东方财富接口 | 备选数据源 |

### 4. 数据质量要求

#### 4.1 完整性要求
- **必填字段完整性**：`open`, `high`, `low`, `close`, `volume` 字段不能为空
- **日期连续性**：交易日数据应连续，不得缺少交易日（停牌除外）
- **数据覆盖度**：每只股票至少应有 135 个交易日的历史数据

#### 4.2 准确性要求
- **价格精度**：价格字段保留 2 位小数
- **成交量精度**：成交量为整数（单位：股）
- **逻辑一致性**：
  - `high` ≥ `max(open, close)`
  - `low` ≤ `min(open, close)`
  - 涨停板验证：涨停日 `close` = `high` 且涨幅 ≥ 9%

#### 4.3 时效性标准
- **更新频率**：每个交易日收盘后（约 17:00 后）更新当日数据
- **数据延迟**：T+1 数据应在 T+1 日 20:00 前完成更新
- **历史数据回溯**：首次使用时需下载至少 200 个交易日的历史数据

## 三、数据获取最佳实践

### 1. 推荐数据获取流程

```
1. 初始化数据源
   └─> 优先使用 Tushare（需配置 Token）
   
2. 获取股票列表
   └─> 使用 Tushare 的 stock_basic 接口
   
3. 批量下载历史数据
   └─> 每批 10 只股票
   └─> 每次获取 365 天数据
   └─> 遵守 Tushare 速率限制（50次/分钟）
   
4. 数据清洗与验证
   └─> 检查必填字段完整性
   └─> 验证价格逻辑一致性
   └─> 补充缺失的交易日（标记为停牌）
   
5. 数据存储
   └─> 保存到本地数据库
   └─> 更新数据更新追踪器
```

### 2. Tushare 接口使用建议

| 接口名 | 用途 | 建议参数 |
|--------|------|----------|
| `stock_basic` | 获取股票列表 | `exchange='', list_status='L'` |
| `daily` | 获取日线数据 | `ts_code` + `start_date` + `end_date` |
| `trade_cal` | 获取交易日历 | `exchange='', start_date` + `end_date` + `is_open='1'` |

### 3. 数据完整性检查清单

- [ ] 所有股票都有至少 135 个交易日的历史数据
- [ ] 每个交易日的 OHLCV 字段完整
- [ ] 价格逻辑正确（high ≥ max(open, close), low ≤ min(open, close)）
- [ ] 数据更新追踪器记录完整
- [ ] 无重复数据
- [ ] 日期格式统一（YYYY-MM-DD）

## 四、总结

### 数据需求对比

| 维度 | top_rise_stock_selector | dragon_leader |
|------|------------------------|---------------|
| **最小历史数据** | 60 个交易日 | 135 个交易日 |
| **推荐历史数据** | 120 个交易日 | 200 个交易日 |
| **必填字段** | OHLCV | OHLCV |
| **可选字段** | amount, pct_chg | amount |
| **最长指标周期** | 60 (EMA60) | 135 (量能均线) |

### 统一数据获取策略

为了同时满足两个策略的需求，建议采用以下统一的数据获取策略：

1. **数据范围**：获取 **365 个交易日** 的历史数据
2. **批量策略**：每次获取 **10 只股票**
3. **数据源**：优先使用 **Tushare**
4. **速率限制**：严格遵守 **50次/分钟** 的限制
5. **数据字段**：获取完整的 OHLCV + amount 字段

这样可以确保：
- ✅ 同时满足两个策略的历史数据需求
- ✅ 留有足够的数据余量用于未来扩展
- ✅ 数据质量和稳定性得到保障
